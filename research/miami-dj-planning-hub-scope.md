# Miami DJ Planning Hub Scope

## Working Brand Note

TC Audio Productions is a working label for this Miami build, not the final brand assumption. Until the final name, domain, logo, and voice are approved:

- Keep new strategy notes focused on the Miami DJ/event planning product, not on TC-specific brand equity.
- Avoid adding new hard-to-verify brand claims, awards, rankings, venue partnerships, or testimonials.
- Write page architecture, CTAs, form fields, and content modules so the brand can be swapped cleanly later.
- Treat "TC" language in existing files as placeholder copy unless it is describing the current repo structure.

## Reference Direction From Loverly

Loverly is useful because it feels like a wedding planning destination, not only a vendor page. The main pattern to borrow is the planning ecosystem:

- Planning tools and checklists.
- Vendor education.
- Inspiration and editorial content.
- Clear next steps for couples.
- Email or inquiry capture after value is delivered.

The Miami DJ site should not become a broad wedding marketplace. It should stay focused on DJ, MC, sound, ceremony audio, lighting, bilingual flow, and planner/venue confidence.

## Core Product Idea

Make the Miami experience feel like:

**A DJ and event production team that helps you plan the room before asking you to book.**

That means the site should sell through guidance, not just through claims. The buyer should be able to answer: What do I need? What could go wrong? What should I ask? What package direction probably fits? What will happen after I inquire?

## Positioning To Preserve

The site should sit between:

- Luxury wedding production: polished visuals, calm confidence, elevated sound and lighting.
- Miami music fluency: bilingual MC, Latin/open-format range, dance-floor credibility.
- Planner-friendly production: timeline discipline, mics, power, load-in, venue notes, backup mindset.

The current line still works as a working framework:

**Clear sound. Polished flow. A room that moves.**

## Modules To Build

### 1. Planning Path Cards

Cards that route users by intent:

- Wedding reception.
- Ceremony + reception.
- Bilingual or Latin wedding.
- Quinceanera/private family celebration.
- Corporate/hotel event.
- Lighting/production support.

Each card should answer "what problem does this path solve?" instead of listing gear first.

### 2. Event Plan Preview

A section showing what the client receives after inquiry:

- Availability review.
- Package direction.
- Timeline notes.
- Music priorities.
- Pronunciation and formalities.
- Planner/venue notes.
- Ceremony audio and outdoor/rain-plan review when relevant.

This is the most important Loverly-inspired idea: planning is part of the perceived value.

### 3. Guided Availability Form Upgrade

Add fields already identified in the backlog:

- MC style.
- Music balance.
- Guest-request policy.
- Must-play songs.
- Do-not-play songs.
- Pronunciation and formalities.
- Outdoor/weather/rain-plan notes.
- Planner or venue contact.

The recommendation box should stay advisory. It should not imply guaranteed pricing or availability.

### 4. Package Recommendation Panel

Connect package cards to the form logic:

- Miami Reception DJ.
- Ceremony + Reception.
- Bilingual Premium Wedding.
- Quince / Private Celebration.
- Corporate / Hotel Event Production.
- Lighting / Production Support.

Show "best for" labels and comparable inclusions. Keep pricing quote-based unless approved.

### 5. Proof Without Fake Proof

Until real proof is approved, use process proof:

- Planning checklist.
- Ceremony audio checklist.
- Must-play/do-not-play worksheet.
- Event packet promise.
- Venue/planner notes.
- Backup-minded language.

Later proof can include real event photos, approved reviews, public profile links, and permissioned planner or venue quotes.
